# FloodSense — Agent Guide

## Quick start

```bash
pnpm install
pnpm run dev        # turbo dev — starts `apps/web` (Vite on port 3001)
pnpm run dev:web    # turbo -F web dev  — web only
pnpm run check-types  # turbo check-types → tsc --noEmit per package
pnpm run build      # turbo build
```

## Monorepo structure

- **pnpm** workspace (v10.20.0 required via `packageManager` in root `package.json`)
- **Turborepo** orchestrates tasks (`turbo.json` at root); `ui: "tui"` (not stream/not web)
- Single active package: `apps/web` (no `packages/` directory — README is stale)

## Web app (`apps/web`)

- **Framework**: React 19, TanStack Router (file-based, v1), TanStack Query (v5), Tailwind CSS v4
- **Dev server**: port **3001** (not 5173 — README is outdated)
- **Entry**: `apps/web/src/main.tsx`
- **Routes**: `apps/web/src/routes/` — file-based; `routeTree.gen.ts` is **auto-generated** by the TanStack Router Vite plugin — do not edit
- **Path alias**: `@/` → `./src/` (configured in both `vite.config.ts` and `tsconfig.json`)
- **CSS**: Tailwind v4 syntax (`@import "tailwindcss"`, no `@tailwind` directives); oklch color tokens; `tw-animate-css` for animations
- **UI components**: shadcn/ui (style: `base-lyra`, icon library: lucide-react) in `src/components/ui/`
- **Theme**: `next-themes` with `ThemeProvider` wrapping root; class-based dark mode
- **PWA**: `vite-plugin-pwa` with auto-update, enabled in dev; run `pnpm -F web generate-pwa-assets` to regenerate PWA icons from `public/logo.png`
- **Env vars** (in `apps/web/.env`):
  - `VITE_SERVER_URL` — backend API base URL (default `http://127.0.0.1:8000`)
  - `VITE_GEOAPIFY_API_KEY` — geocoding API key

## API contract

The frontend calls `POST {VITE_SERVER_URL}/predict` with `{ lat, lng }` JSON body.
Returns `FloodPredictionResponse` with `baseline_rf`, `ensemble` predictions, and `extracted_values`.
No server code lives in this repo — it's expected to run separately.
